How it works

Payment rails (Stripe, x402, Visa AP2, Skyfire) handle how agents pay. OpenTrustSeal answers the question that comes first: should this agent pay this merchant at all?

1

Six signal categories, zero guesswork

Domain age, SSL/TLS, DNS security, content analysis, reputation databases (Spamhaus, SURBL, Google Safe Browsing), and identity verification. Each signal is verifiable public evidence. The scoring model is versioned and every score is reproducible from stored raw data.

2

Signed evidence, not opaque scores

Every response is cryptographically signed with Ed25519. Agents verify the bundle without trusting us. Payment protocols can cite our verdicts without putting us on their critical path. The evidence bundle is the product; the trust score is a convenience summary over it.

3

100,000+ domains scored

The registry covers the Tranco top-100K with longitudinal trust profiles. A six-tier fetch ladder (direct, headless Chrome, residential proxy, Wayback Machine, protocol probes) ensures comprehensive content analysis even for sites behind aggressive bot protection.

For developers

Verify any domain before a transaction. Signed with Ed25519 so results cannot be forged. Works today with LangChain, CrewAI, OpenAI tools, and any agent framework that can make an HTTP call.

# Check any domain before your agent pays
# GET https://api.opentrustseal.com/v1/check/merchant.com

import httpx

r = httpx.get("https://api.opentrustseal.com/v1/check/chewy.com")
data = r.json()

# data["trustScore"] -> 77
# data["recommendation"] -> "PROCEED"
# data["brandTier"] -> "well_known"
# data["crawlability"] -> "ok" or "blocked"
# data["signals"] -> 6-category evidence breakdown
# data["flags"] -> [WELL_KNOWN_BRAND, ...]
# data["checklist"] -> actionable improvements
# data["signature"] -> Ed25519 cryptographic proof
# data["jurisdiction"] -> country + legal framework + cross-border risk

if data["recommendation"] == "DENY":
    print("Refusing:", data["reasoning"])
elif data["recommendation"] == "CAUTION":
    confirm_with_user(data["reasoning"])
